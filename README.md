# AI-phishing-email-system

本專案是一個基於 AI 的釣魚郵件偵測系統，旨在透過機器學習與資安前處理技術，提高對惡意郵件的識別準確度。

## 一、 系統核心架構

本專案拆分為三個獨立但可串接的模組，確保開發過程不互相依賴：

1.  **前端輸入層**： 接收原始郵件文本的 Web API 與使用者介面 (UI)。
2.  **資安前處理層 (Security Pre-processing)**： 負責清洗雜訊、提取惡意指標 (IOCs)，並執行反混淆（如解析 Base64 或 URL 編碼）。
3.  **AI 推論層 (AI Inference)**： 基於 Random Forest 演算法的機器學習模型，輸出威脅機率與分類結果。

---

## 二、 敏捷開發流程 (Agile Workflow)

### 第一階段：打通端到端資料流 (Pipeline Construction)
*   **目標**： 建立最小可行性產品 (MVP)，確保系統連通性。
*   **具體執行**：
    *   定義標準 JSON 資料交換格式。
    *   架設前端介面。
    *   實作 Python 腳本接收前端字串、執行基礎去雜訊，並呼叫模型回傳結果。

### 第二階段：模型微調與資安特徵工程 (Core Engineering)
*   **目標**： 提升模型準確率，強化資安領域特徵。
*   **具體執行**：
    *   將開源資料集按 80/20 比例切分為訓練集與驗證集進行訓練。
    *   在處理層加入深度解析邏輯，自動識別並還原隱藏的 URL 或 Base64 惡意負載 (Payload)。

### 第三階段：對抗性驗證與系統優化 (Red Teaming & Validation)
*   **目標**： 檢驗系統強健力並完成最終交付。
*   **具體執行**：
    *   凍結模型權重，啟動 AI 紅隊演練 (Red Teaming)。
    *   生成對抗性變異文本（如同義詞替換、符號混淆）攻擊模型。
    *   紀錄偽陽性 (False Positives) 與偽陰性 (False Negatives) 速率，並進行最終部署展示。

## 四、 開發規範與紀律

1.  **版本控制**： 統一使用 Git。`main` 分支保持穩定，新功能於獨立分支開發後經確認再併入。
2.  **環境一致性**： 統一使用 `environment.yml` 或 `requirements.txt` 配置，確保套件版本完全一致，避免環境相容性問題。
